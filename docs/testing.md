# Pengujian Sistem - Black Box Testing

## Pendahuluan

Pengujian sistem dilakukan menggunakan metode Black Box Testing untuk memverifikasi fungsionalitas sistem dari perspektif pengguna. Metode ini berfokus pada validasi input dan output sistem tanpa memperhatikan struktur internal kode program. Pengujian disusun berdasarkan pembagian increment dan peran aktor yang berinteraksi dengan sistem.

Setiap test case dilengkapi dengan:

-   **Skenario Pengujian**: Berisi pre-condition dan langkah-langkah pengujian secara sistematis
-   **Test Data**: Data konkret yang digunakan dalam pengujian
-   **Skenario Negatif**: Ditambahkan setelah setiap skenario positif terkait untuk menguji validasi dan penanganan kesalahan

## Kesimpulan Pengujian

Pengujian sistem menggunakan metode Black Box Testing telah dilaksanakan secara komprehensif terhadap seluruh fungsionalitas yang diimplementasikan pada Increment 1 dan Increment 2. Pengujian dilakukan berdasarkan perspektif masing-masing aktor yaitu Bagian Inventori, Bagian Produksi, Kasir, Admin, Pengguna Baru, dan Semua Pengguna.

Total skenario pengujian yang disusun mencakup **184 test case** yang terdiri dari:

-   **Increment 1 — Modul Inti Operasional:**

    -   Bagian Inventori: 51 test case (TC-001 s.d. TC-051)
    -   Bagian Produksi: 13 test case (TC-052 s.d. TC-064)
    -   Kasir: 18 test case (TC-065 s.d. TC-082)

-   **Increment 2 — Modul Pendukung Operasional:**
    -   Admin: 48 test case (TC-083 s.d. TC-130)
    -   Pengguna Baru: 8 test case (TC-131 s.d. TC-138)
    -   Bagian Inventori: 23 test case (TC-139 s.d. TC-161)
    -   Kasir: 13 test case (TC-162 s.d. TC-174)
    -   Bagian Produksi: 10 test case (TC-175 s.d. TC-184)

Setiap test case dilengkapi dengan:

-   **Skenario pengujian** yang mencakup pre-condition dan langkah-langkah sistematis
-   **Test data** konkret berdasarkan implementasi aktual sistem
-   **Skenario negatif** untuk menguji validasi input, penanganan error, dan kondisi batas

Skenario pengujian disusun berdasarkan peninjauan kode sumber yang mencakup validasi input pada komponen Livewire, aturan routing dan middleware, serta logika bisnis yang diimplementasikan dalam sistem. Setiap negative test case mencerminkan validasi yang benar-benar diterapkan pada kode program, antara lain:

-   **Login dan autentikasi:** Validasi field kosong, password salah, akun belum diaktifkan, akun dinonaktifkan, rate limiting (5 percobaan)
-   **Tambah data (create):** Validasi field wajib kosong, format input tidak valid, data duplikat (unique constraint), nilai di bawah minimum, format file tidak sesuai, ukuran file melebihi batas
-   **Ubah data (update):** Validasi nama duplikat dengan pengecualian ID sendiri (Rule::unique()->ignore), format data tidak valid
-   **Hapus data (delete):** Perlindungan penghapusan data yang masih digunakan (satuan digunakan bahan baku, jenis biaya digunakan produk, peran memiliki pengguna)
-   **Proses transaksi dan operasional:** Pembayaran kurang dari minimum (50% untuk pesanan, 100% untuk siap beli), stok habis, bahan baku tidak mencukupi untuk produksi, kuantitas melebihi ekspektasi belanja
-   **Sistem poin:** Poin bukan kelipatan 10, poin melebihi saldo tersedia
-   **Pengaturan hak akses:** Pencegahan deaktivasi akun sendiri, batas maksimal pengguna per peran, pengiriman ulang undangan pada akun yang sudah aktif
-   **Aktivasi akun:** Token tidak valid, token kedaluwarsa, akun sudah diaktivasi, password tidak memenuhi kriteria (min 8 karakter, harus huruf dan angka, konfirmasi cocok)

Hasil pengujian menunjukkan bahwa seluruh skenario pengujian berhasil dilaksanakan dengan output yang sesuai dengan ekspektasi. Sistem dapat memvalidasi input, memproses data dengan benar, menghasilkan output yang sesuai, serta menampilkan pesan kesalahan yang informatif ketika terjadi kondisi tidak valid.

Berdasarkan hasil pengujian, dapat disimpulkan bahwa Sistem Pawon3D telah berfungsi sesuai dengan spesifikasi kebutuhan dan siap digunakan untuk mendukung operasional toko kue.
